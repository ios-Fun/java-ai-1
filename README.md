#  短信发送硬件的通信方式选型
  * 串口
    * 优点：调试、部署、开发，效率高
    * 缺点：不如网口稳定
    * 目前选择的方式
  * 网口
    * 配置、开发；更繁琐
# Java串口开发的集成方式选型
  * RXTX
    * 要导入rxtxSerial.dll库
    * 引入RXTXcomm.jar到工程
    * 特点：引入dll库，相对繁琐
  * JSerialComm
    * maven引入JSerialComm依赖就可以
    * 特点：方便
  * purejavacomm
    * 当前选择的方式，和JSerialComm类似
    * maven引入purejavacomm依赖就可以，库引入了c方法，不用导入dll
# 调试工具
  * 1. 查看服务器的端口， 通过控制面板的设备管理器查看，端口一般是COM1、COM3这种
    * 如果服务器没有串口的硬件接口，需要额外的USB转串口的硬件和安装驱动
  * 2. 下载运行串口调试工具，如sscomm.5.13.1.exe
  * 3. 设备通过串口线与服务器连接后，打开串口调试工具
    * 选择端口号：如COM1;
    * 选择波特率：一般是115200或9600
    * 点击“打开串口”按钮
  * 4. 测试内容输入：手机号:0:内容；如 18621909785:0:当前有告警请及时查看 点击发送，能收到短信或调试工具显示返回数据，表示设备和通信功能正常
    * 注意：中间的冒号用英文输入法
# Java开发
  * 串口通信是异步的，不方便做到同步的返回发送成功或失败的结果
  * 串口发送消息最好是单线程下进行
    * 将http的多线程的的请求的对象，放在ConcurrentLinkedQueue，并用一个线程SendSmsTask去不断循环处理发送请求
    * 这样单线程SendSmsTask去进行串口数据的发送，并可以在线程内自行控制发送间隔
  * 用一个线程ReceiveSmsTask去处理接收数据
  * 主要有两个变化参数，放在nacos配置中
    * serial.portName，如COM1
    * serial.rate，如115200
    * serial.isEnable；短信发送功能是否开启，可能一些业务情况下需要某个时间段关闭
  * 提供接口给其他服务调用
    * 参数1，phone数组
    * 参数2，content，发送内容
